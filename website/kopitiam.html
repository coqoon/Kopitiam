Kopitiam

Kopitiam is an Eclipse plugin for certifying full functional correctness of Java programs using a higher-order separation logic. Kopitiam extends the Java IDE with an interactive environment for program verification, powered by the general-purpose proof assistant Coq. Moreover, Kopitiam includes a feature-complete development environment for Coq theories, where users can define program models, and prove theorems required for the program verification.

Coq perspective (click to enlarge): on the left bottom the outline, in the middle the Coq editor and problems view, on the right the goal viewer. The Coq editor indicates with a green background the commands processed by Coq, the red squiggly line indicates an error (no command 'eapply xist' exists) and the tooltip shows all possible completions with the prefix 'xist'.

The Java perspective (clieck to enlarge): Java code interleaved with specifications and proof script using antiquotes. Similar to the Coq perspective, the green background indicates commands processed by Coq. On the right the goal viewer shows the current proof state.

Installation Instructions

  • Coq - currently Coq-8.3 (direct link) is required
  • Charge! - Java semantics, higher-order separation logic and tactics in Coq
    • Unpack the tarball and use 'make' to compile
  • Eclipse - currently a 32bit version of Indigo (3.7) (direct link) is required
  • Start Eclipse
  • Click on the "Help -> Install New Software" menu item
  • Enter "http://itu.dk/people/hame/kopitiam/" as update site
  • Select Kopitiam and follow the dialogs
  • Restart Eclipse
  • Go to Preferences -> Kopitiam and configure the path to coqtop and the loadpath to your downloaded Charge!

Eclipse Example Projects

These example projects also include the proof certificate file (ListJava.v/FactorialJava.v), which is generated by Kopitiam and can be checked automatically using Coq and Charge!.

  • Linked List implementation with add, in-place list reversal and a recursively implemented length method
  • Factorial implementation using a static method which calls itself recursively
